Overview›Tools›Device Search App
Tool Detail — 01 of 05
Device Search App
Windows desktop search utility with a Python interface and high-speed Rust parsing engine — searches multiple years of warehouse data in under 2 seconds.
❌ The Problem

Purchase records were stored across multiple large Excel files — one per year, each with thousands of device entries. Finding a single device meant manually opening the right year's file, scrolling through sheets, and matching entries one by one. As data grew over years, this became noticeably slower. Urgent lookups during a sale or reconciliation wasted time every single day.

✅ The Solution

A standalone Windows desktop utility with a Python interface and Rust parsing engine that loads all data into a local database and returns results in under 2 seconds. Also integrated into Excel via VBA: when a sales sheet is prepared, the macro queries the database for every device, confirms inventory existence, and flags any price mismatch in red — without opening the app manually.

What the App Does
  • Searches all years of purchase data simultaneously — results in under 2 seconds on any record count
  • Partial and exact matching across all fields: brand, model, condition, serial number, IMEI, warehouse location
  • Loads data into a local analytical database on first launch — searches run on the database, not Excel files directly
  • Auto-detects file changes every 30 seconds and refreshes automatically
  • Progressive loading — search available immediately while additional data still loads in the background
  • Runs in the system tray — always ready without keeping a window open
  • Export search results to Excel or CSV with one click
Excel-to-Database Integration Unique
  • A purpose-built data integration layer connects Excel workbooks directly to the local analytical database engine
  • Supports batch insertion of 2,500 records at a time and multi-table SQL queries — reducing crashes and manual handling on high-volume files
  • When a sales sheet is prepared (e.g. up to 1,000 devices across 10 daily bills), the VBA macro queries the database for every device automatically
  • Confirms each device exists in inventory and that the sale price matches the purchase record
  • Flags price mismatches in red — directly inside the sales sheet before finalising
  • This is the post-payment confirmation step — verifying delivery matches what was paid
How It Was Built AI-assisted
  • Python interface with a high-speed Rust parsing engine — built with AI assistance in VS Code
  • Uses a local analytical database (DuckDB) for fast queries on large datasets
  • Compiled to standalone Windows .exe via PyInstaller — no Python needed on any machine
  • Multi-threaded: interface stays responsive while data loads or refreshes
  • GUI built with CustomTkinter; SQLite for settings persistence between sessions

Tool Detail — 02 of 05
Price Audit & Reconcile App
Desktop application with three built-in tools for stock reconciliation and price verification — built because manual comparison before payment was too slow and too unreliable.
❌ The Problem

There was no structured way to confirm a supplier's delivery matched the original order in quantity, condition grade, and price before payment. Short deliveries, condition changes, and billing errors could only be caught through slow manual Excel comparison. Rate list updates were equally hard to evaluate — manually comparing two large files across hundreds of model-condition combinations was unreliable and time-consuming.

✅ The Solution

A three-in-one desktop app: the Reconcile tool compares what was ordered against what arrived and flags every difference; the Ratecard Analyser shows exactly what changed between an old and new price list; the Audit tool scans a single file for internal pricing inconsistencies. All three run in seconds via drag-and-drop.

Tool 1 — Reconcile Files
  • Drag and drop delivery file and original order file — comparison runs immediately
  • Identifies: missing devices, quantity differences, condition grade changes, price discrepancies
  • Exports full report to Excel — summary sheet plus every flagged issue listed clearly
  • Smart column auto-detection — adapts to different file formats without manual mapping
  • Case-insensitive, normalised matching handles inconsistent naming in real supplier files
Tool 2 — Ratecard Analyser
  • Load old and new supplier price list files — changes identified automatically
  • Shows every change per model and condition grade: previous rate, new rate, exact difference
  • Flags newly added combinations and items removed from the list
  • Export comparison report to Excel for records and management review
Tool 3 — Single File Audit
  • Scans one file for internal inconsistencies — same model and condition with different prices
  • Catches supplier data entry errors before the file enters the workflow
  • Runs in seconds — used on every new file before accepting it
How It Was Built AI-assisted
  • Rust (Tauri framework) backend, React + Vite + TailwindCSS frontend — AI-assisted
  • Compiles to native Windows .exe — fast, lightweight, no installation needed
  • SheetJS for Excel file reading; column mappings saved between sessions
  • Lazy loading and file preloading for smooth performance on large files
  • Vite frontend communicates via IPC triggers; custom Rust excel engine
Overview›Tools›Gmail File Collector
Tool Detail — 03 of 05
Gmail File Collector
Automated script replacing 30+ minutes of daily manual file downloading — processes 50–100 supplier email attachments every day with zero manual steps.
❌ The Problem

The business receives 50–100 supplier file attachments on Gmail every day — order files, invoice files, and stock detail files spread across dozens of email threads. Previously, someone had to open Gmail, manually download each attachment, decide which folder it belonged to, and move it there. This was 30+ minutes of daily work adding zero value and regularly leading to files saved in the wrong location or downloaded twice.

✅ The Solution

A Google Apps Script that runs on demand. It scans Gmail for all emails with the designated label, downloads only Excel attachments, reads the email subject line to detect the warehouse code, creates the correct subfolder automatically, and saves each file in the right location — skipping anything already downloaded. A separate cleanup script removes processed files once used.

What It Does
  • Scans Gmail for all emails with the designated supplier label
  • Downloads only Excel file attachments — ignores PDFs, images, unrelated files
  • Sorts files into correct folder category based on file type
  • Reads email subject line to detect warehouse code — creates correct subfolder per warehouse
  • Skips any file already downloaded — no duplicates, no overwriting, ever
  • Built-in logging for every execution — full record of what was downloaded or skipped
  • Separate cleanup script removes processed files once used, keeping folders clean
Technical Approach
  • Written in Google Apps Script (JavaScript) — runs within Google Workspace, no server needed
  • Uses Gmail API and Google Drive API v3 to access email and save files
  • Set-based duplicate checking — fast and reliable across 100+ files per run
  • Folders created only when files exist — no empty folder clutter in Drive
  • Subject line parsing handles multiple warehouse code formats automatically

Full Daily Workflow — 8 Steps
Receive Supplier Lot File
A pricing reference file arrives for each purchase lot — contains product codes, brand, model, condition grade, base price, agreed pricing structure (flat rates, percentage discounts, or adjusted margins), and the final agreed rate per unit.
Gmail Auto-Download(50–100 Order Files)
The Gmail automation script downloads all supplier order file attachments for the day — each file contains product codes, product title, brand, grade, base price, and final value. Sorted by file type and warehouse automatically. Duplicates skipped.
Rate Calculation & Power Query Processing
Agreed pricing formulas (accommodating flat rates, percentage discounts, or adjusted margins) are applied to calculate the final net purchase rate per unit. Power Query then merges all order files, cleans the data, and produces a master workbook: one sheet per lot, a merged data sheet, and a summary showing total quantity, total amount, condition breakdown, and brand split. Example: one run covers 11 lots and 1,021 phones.
Payment Released
Payment is confirmed and made to the supplier based on the agreed order rate and total. This happens before receiving the detailed invoice and stock files — the supplier sends those after payment is received on their end.
Receive Invoice and Stock Detail Files
After payment is received by the supplier, they send invoice files (product code, price, tax breakdown, total per unit) and stock detail files (warehouse serial number, IMEI, brand, model, condition grade). These are auto-downloaded via the same Gmail script.
3-Way Stock Reconciliation
Compare three data sources: what was originally ordered, what actually arrived, and what was charged. Short-delivered units, condition grade changes, and pricing differences all identified. Any discrepancy is documented and escalated for correction against the payment already made.
Update Master Yearly Database
Every device from the batch is entered into the master database — IMEI, warehouse serial number, condition grade, purchase amount, warehouse location (Noida or Bilaspur), and batch reference. Single source of truth for all inventory across the business.
Post-Payment Delivery Confirmation
A final check comparing the payment made against the confirmed delivery and invoice data. The Excel-to-database integration queries every device in the sales batch, confirms each unit is recorded correctly in inventory, and flags any discrepancy between what was paid and what was received. Tracked and resolved if any difference is found.

Reporting Deliverables
  • Daily stock summaries — quantity per batch, condition breakdown, brand split, total amount
  • Weekly, 15-day, and monthly batch performance reports covering sales volume and stock movement
  • Customer sales sheets listing every device sold: condition, amount, serial number, IMEI — auto-generated per sale
  • Batch payments, open balances, and customer receipts tracked for up to 1,000 devices across 10 daily sales bills
  • Management pivot tables: stock by condition, brand, warehouse, seller
Other Key Contributions
